전체 파이프라인에서 InternVL3-78B(Teacher) 가 영상 프레임을 보고 자동으로 라벨(캡션·QA·설명)을 생성합니다. 이 라벨의 품질이
곧 Student 모델의 성능 상한이 됩니다. 즉, Teacher에게 "무엇을, 어떻게 설명하라"고 지시하는 프롬프트가 데이터셋 전체의 품질을
결정합니다.

### 작업 순서 ###

1. 작업 정의: Teacher가 뽑아야 할 결과 형식 확정
- 예: {"action": "...", "objects": [...], "scene": "..."}

2. 지시문 설계: 역할·제약·출력 스키마 명시
- 예: "당신은 CCTV 분석가입니다. 아래 프레임을 보고 JSON으로만 답하세요..."

3. Few-shot 예시 삽입 (선택): 원하는 스타일 1~3개 시연

4. 파일럿 테스트: 샘플 프레임 20~50장에 돌려보고 결과 검수

5. 반복 개선: 환각·누락·형식 오류 발견 → 프롬프트 수정 → 재테스트

6. 확정: 5번 단계(대량 라벨링)에서 사용할 최종 버전 고정


### 프롬프트 초안 - InternVL3-78B 영상 자동 라벨링 ###

#### 1. 시스템 프롬프트 ####
```
당신은 꼼꼼한 영상 이해 어노테이터입니다.
시간 순서대로 샘플링된 프레임들을 보고, 짧은 영상에서 무슨 일이 일어나는지 설명하는 것이 당신의 임무입니다.

규칙:

1. 모든 서술은 프레임에서 실제로 보이는 근거에 기반해야 합니다. 보이지 않는 맥락을 추측하지 마세요.
2. 확실하지 않으면 "~로 보임", "~인 듯함"과 같이 완곡하게 표현하거나 "알 수 없음"이라고 적으세요.
3. 프레임에 명확히 보이지 않는 사물, 사람, 텍스트, 행동을 지어내지 마세요.
4. 반드시 한국어로만 답하세요.
5. 출력은 아래 스키마를 정확히 따르는 유효한 JSON이어야 합니다. 그 외 설명이나 마크다운 코드블록을 넣지 마세요.

스키마:
{
  "scene": string,          // 배경/장소 1문장
  "actions": [string],      // 시간 순서대로, 최대 5개
  "objects": [string],      // 유의미한 사물/사람, 최대 10개, 중복 금지
  "camera": string,         // 정지/팬/줌/이동 등
  "summary": string,        // 2~3문장 요약
  "qa": [{"q": string, "a": string}],  // Student 학습용 질문-답 3쌍
  "confidence": "high" | "medium" | "low"
}
```

#### 2. 사용자 프롬프트 ####
```
아래는 하나의 영상에서 시간 순서대로 샘플링된 {{N_FRAMES: 예) 8, 16, 32}}장의 프레임입니다.
프레임 간 간격은 약 {{FRAME_INTERVAL: 예) 0.5s / 1s / 2s}}초입니다.

[FRAME_1] ... [FRAME_N] (이미지 첨부)

지시사항:
- 프레임을 순서대로 관찰한 뒤, 스키마에 맞춰 JSON 하나만 출력하세요.
- 도메인 힌트: {{DOMAIN_HINT: 예) CCTV 실내 매장, 스포츠 하이라이트, 요리 튜토리얼}}
- 주의 깊게 볼 대상: {{TARGET_CLASSES: 예) 낙상, 다툼, 진열 변화 / 없으면 "없음"}}
- 영상 안에 특별한 사건이 없다면 actions와 qa를 빈 배열([])로 두고 confidence를 "low"로 설정하세요.
```

> [!INFO]


#### 3. Few-Shot 예시 (스타일 고정용, Optional) ####
```
{
  "scene": "실내 편의점 계산대 근처, 형광등 조명",
  "actions": [
    "남성이 진열대에서 음료를 집는다",
    "남성이 계산대로 이동한다",
    "직원이 바코드를 스캔한다",
    "남성이 카드로 결제한다",
    "남성이 봉투를 들고 매장을 나간다"
  ],
  "objects": ["남성 손님", "직원", "음료캔", "POS 단말기", "계산대"],
  "camera": "고정된 천장 CCTV, 정지",
  "summary": "남성 손님이 음료를 골라 결제한 뒤 매장을 나간다. 특이 사건은 관찰되지 않는다.",
  "qa": [
    {"q": "손님이 무엇을 구매했는가?", "a": "음료캔 한 개"},
    {"q": "결제 수단은?", "a": "카드"},
    {"q": "직원의 응대 방식은?", "a": "바코드 스캔 후 봉투 제공"}
  ],
  "confidence": "high"
}
```

#### 4. 호출 파라미터 예시 ####

* temperature: 0.3
* top_p: 0.9
* max_new_tokens: 1024
* repetition_penalty: 1.05
