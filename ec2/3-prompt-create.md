  전체 파이프라인에서 InternVL3-78B(Teacher) 가 영상 프레임을 보고 자동으로 라벨(캡션·QA·설명)을 생성합니다. 이 라벨의 품질이
  곧 Student 모델의 성능 상한이 됩니다. 즉, Teacher에게 "무엇을, 어떻게 설명하라"고 지시하는 프롬프트가 데이터셋 전체의 품질을
  결정합니다.

  ### 작업 항목 ###
  
  1. 작업 정의: Teacher가 뽑아야 할 결과 형식 확정
    - 예: {"action": "...", "objects": [...], "scene": "..."}
  
  2. 지시문 설계: 역할·제약·출력 스키마 명시
    - 예: "당신은 CCTV 분석가입니다. 아래 프레임을 보고 JSON으로만 답하세요..."
  
  3. Few-shot 예시 삽입 (선택): 원하는 스타일 1~3개 시연
  
  4. 파일럿 테스트: 샘플 프레임 20~50장에 돌려보고 결과 검수
  
  5. 반복 개선: 환각·누락·형식 오류 발견 → 프롬프트 수정 → 재테스트
  
  6. 확정: 5번 단계(대량 라벨링)에서 사용할 최종 버전 고정
